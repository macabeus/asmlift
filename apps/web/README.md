# asmlift webapp

The public site — **fully client-side**, the [`@asmlift/core`](../../packages/core/README.md)
pipeline bundled for the browser (its enforced zero-dependency purity is what makes this
possible). No server, no queue, nothing leaves the tab. Two views of one app, switched from the
header:

- **Playground** — paste one function's assembly, get C back. CodeMirror editor with per-target
  examples (real committed disassembly from `packages/core/test/corpus/` — load-bearing, see
  [the corpus note](../../packages/core/README.md#tests)), a C / C++ / Pascal output selector, gap
  diagnostics, and the **Pipeline** tab (the `decompileTraced` stage timeline with per-stage IR
  dumps and pattern before/after cards).
- **Benchmark** — the m2c-vs-asmlift report (Overview / Function Explorer / Gap Analysis /
  Methodology) over the committed benchmark dataset. **Lazy-loaded**: its ECharts-plus-dataset
  chunk only downloads on the first visit. A function's "Open in playground" button hands its input straight
  to the editor (in-app, no round-trip).

Navigation state lives in typed query params, so every view is a shareable URL — the active
view, a playground permalink, and the full explorer state down to an open function detail.
`src/App.tsx` is the shell that owns the header and the view switch.

### Layout

```
src/
  App.tsx  main.tsx  index.css      the shell + entry
  assets/  data/                    logo, committed summary.json
  pages/
    playground/                     Playground-only (editor, ranking worker, pipeline, examples)
    benchmark/                      Benchmark-only (report views, charts, results.json)
  shared/
    components/                     used by BOTH pages (e.g. CodeBlock)
    utils/                          used by both (e.g. permalink.ts)
```

The boundaries are enforced by **dependency-cruiser** (`pnpm run check-deps`, also a CI step):
pages must not import each other (share via `src/shared/`), a page is reachable from outside only
through its entry component (`playground/Playground`, `benchmark/Benchmark`), and `src/shared/` must
not depend on a page. See `.dependency-cruiser.cjs`.

```bash
pnpm install       # from the repo root (workspace member)
cd apps/web
pnpm run dev       # local dev server
pnpm run build     # two tsc passes (src, then tests + vite.config) + vite build
```

Deploys to GitHub Pages at `/asmlift/` (see [deploy notes](../../README.md#deploy)). The Benchmark
dataset (`src/pages/benchmark/data/results.json`) and the playground's footer summary
(`src/data/summary.json`) are both regenerated by `pnpm run bench:merge` and committed.

Unlike the rest of the monorepo (MIT), this app is **GPL-2.0-only** — it bundles the GPL
`agbcc` compiler. See [`NOTICE.md`](NOTICE.md) for why and where the corresponding source lives.
